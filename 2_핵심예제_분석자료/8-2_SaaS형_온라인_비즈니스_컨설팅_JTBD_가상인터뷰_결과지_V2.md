### 1. JTBD 요약 카드 (Job Story)

8개의 핵심 페르소나에 대한 심층 인터뷰 결과, MVP가 해결해야 할 가장 중요한 Job들은 명확하게 Q1(핵심 혁신 기회) 영역에 집중되었습니다.

| **구분** | **내용** |

| **Desired Outcome (… so I can …)** | '성급한 확장(Premature Scaling)'이라는 **실패 공식을 반복하지 않고** 확신을 갖고 개발에 집중하거나, 낭비 없이 방향을 수정(Pivot)할 수 있도록 |

| **Current Solutions / Workarounds** | Notion(가설 관리) + Excel(간단한 재무) + 고객 인터뷰(수동) |
| **Switch Triggers / Barriers** | **Trigger:** 수동으로 관리하던 가설 노트가 100개가 넘어가며 엑셀과 연동이 안 되기 시작함<br>**Barrier:** 이미 Notion과 Excel에 익숙해져 있어 데이터 이전(Migration)이 불안함 |
| **Evidence (근거)** | "지난번엔 제가 만든 제품과 사랑에 빠졌어요. 이번엔 '문제'와 사랑에 빠지려고요."<br>"사업계획서 100페이지보다, '이거 하지 마라'는 적신호 1개가 더 중요합니다." |
| **Priority (AOS / DOS)** | **AOS = 3.0**, **DOS = 2.7** (Q1 핵심 혁신 기회) |
| **Notes (추가 메모)** | MVP 1순위 가치 타겟. 'PMF 대시보드', '가설-검증' 기능이 핵심. |

| **구분** | **내용** |
| --- | --- |
| **Persona / Segment** | **스위처 (Switcher)** – '이대표' (LivePlan 유료 구독 후 이탈, SaaS 창업가) |
| **Situation (When …)** | LivePlan을 결제해 QBO 연동으로 완벽한 재무 계획을 만들었지만, 막상 팀원들은 아무도 그 툴에 접속하지 않을 때 |
| **Job Statement (… I want to …)** | LivePlan의 '재무 계획'과 팀원들이 실제 일하는 Notion/Jira의 **'실행 전략'을 하나의 툴에서 연동/관리**하고 싶다 |
| **Desired Outcome (… so I can …)** | '계획'과 '실행'이 분리되어 대표만 보는 유령 계획서가 되고, 결국 다시 Excel로 회귀하는 문제를 피하고 싶다 |
| **4 Forces** | **Push:** **"실행과 재무의 단절"**. 재무팀(LivePlan)과 개발팀(Jira)이 다른 데이터를 봄<br>**Pull:** 'Traction Roadmap'과 '재무 대시보드'가 통합된 솔루션에 대한 기대<br>**Habit:** LivePlan(재무) + Notion(전략) + Excel(취합)을 매달 수동으로 비교하는 습관<br>**Anxiety:** "새 툴을 도입해도 어차피 Jira/Notion 연동이 완벽하지 않으면 또 하나의 사일로(Silo)가 될 것이다." |
| **Current Solutions / Workarounds** | LivePlan(유료) + Notion(유료) + Google Sheets(수동 취합) |
| **Switch Triggers / Barriers** | **Trigger:** 투자자 미팅에서 "재무 계획상 마케팅비는 왜 늘었는데, 실제 실행한 가설은 없죠?"라는 질문에 데이터가 분리되어 답변을 못한 순간 (Firing LivePlan)<br>**Barrier:** 기존 LivePlan과 Notion에 쌓인 데이터를 모두 이전해야 하는 부담 |
| **Evidence (근거)** | "LivePlan은 훌륭한 '재무 계획' 툴이지만, '혁신 실행' 툴은 아니었어요."<br>"매주 월요일마다 Notion의 실행 로드맵과 LivePlan의 재무 예측을 엑셀에서 비교하는 게 제 일입니다. 이게 맞나요?" |
| **Priority (AOS / DOS)** | **AOS = 3.0**, **DOS = 2.4** (신규 검증) |
| **Notes (추가 메모)** | 기존 가설 3('실행-재무 단절')을 명확히 검증함. '계획'과 '실행'의 연동이 핵심 차별화 포인트. |

| **4 Forces** | **Push:** '매출원가', 'CAC', 'LTV' 같은 용어를 볼 때마다 느끼는 좌절감. "나는 이런 걸 할 사람이 아닌가"라는 자괴감.<br>**Pull:** '초보자 가이드', '왕초보도 5분 완성' 같은 쉽고 친절한 메시지.<br>**Habit:** 어려운 부분은 건너뛰거나, 자녀에게 "이것 좀 해달라"고 부탁하는 습관.<br>**Anxiety:** "내가 뭔가 잘못 입력해서 사업 전체를 망치는 계산을 하게 될까 봐 두려움." |
| **Current Solutions / Workarounds** | 한컴오피스 한글(HWP)에 무작정 줄글로 아이디어 나열하기, 포기하기 |
| **Switch Triggers / Barriers** | **Trigger:** 유료 SaaS 툴의 7일 무료 체험 기간 동안 버튼 3개 눌러보고 그대로 종료했을 때.<br>**Barrier:** 대부분의 SaaS 툴이 젊은 IT 스타트업 기준으로 설계되었다는 고정관념. |
| **Evidence (근거)** | "설명서가 더 어려워요. 그냥 1번엔 이거, 2번엔 이거 쓰라고 알려주면 좋겠어요."<br>"제가 원하는 건 대단한 기능이 아니라, 끝까지 포기하지 않게 도와주는 친절함이에요." |
| **Priority (AOS / DOS)** | **AOS = 4.0**, **DOS = 3.2** (Q1 핵심 혁신 기회) |
| **Notes (추가 메모)** | 모든 사용자의 이탈을 막는 핵심 경쟁력. '쉽고 직관적인 UX'는 선택이 아닌 필수. |

### 3. 종합 결론 및 MVP 전략 제안

JTBD 가상 인터뷰 결과는 AOS-DOS 매트릭스 분석(첨부 7)의 **Q1(핵심 혁신 기회) 영역이 실제 고객의 가장 고통스러운 문제임을 명확히 재검증**합니다. 따라서 MVP는 다음 세 가지 핵심 Job의 해결에 모든 자원을 집중해야 합니다.

1.  **Job 1: 과제 통과 (Pass the Test)**
    - **Pain:** 정부 지원 탈락, 은행 대출 거절에 대한 공포.
    - **Evidence:** 김예비("합격하는 계획서"), 박사장("은행에서 원하는 서류").
    - **MVP Solution:** **정부/은행 공식 양식 100% 호환 템플릿**, AI 기반 초안 자동 완성, 제출용 PDF/HWP 원클릭 내보내기 기능.

2.  **Job 2: 실패 회피 (Avoid the Failure)**
    - **Pain:** 과거 실패 트라우마, 가설과 재무의 단절, 객관적 검증 부재.
    - **Evidence:** 최민혁("실패 공식 반복 방지"), 한서윤("숫자의 근거"), 이대표("계획과 실행의 단절").
    - **MVP Solution:** **핵심 가설-재무 연동 대시보드**, PMF(제품-시장 적합성) 진단 프레임워크, 시나리오별 재무 시뮬레이션 기능.

3.  **Job 3: 장벽 극복 (Overcome the Blocker)**
    - **Pain:** 보안 불신, 복잡한 UX, 재무 공포증.
    - **Evidence:** 윤박사("IP 유출 불안"), 박은숙("용어의 장벽"), 오다빈("엑셀 공포").
    - **MVP Solution:** **설치형(On-premise) 옵션 제공**, **단계별 마법사(Wizard) UI**, **핵심 변수 기반 재무 자동화 엔진**. 이들은 단순 기능이 아닌, 특정 고객 그룹을 시장으로 유입시키기 위한 **전제 조건(Prerequisite)**입니다.

이 세 가지 핵심 Job을 해결하는 제품만이 초기 시장의 강력한 수요(Volume)와 충성도 높은 고객(Value)을 동시에 확보하며 지속 가능한 성장 기반을 마련할 수 있을 것입니다.
