### 1. 문제해결(=문제정의 중심) 방법론 구조

| 단계 | 목적 | 주요 활동·질문 | 도구·프레임워크 | 산출물 | 비고 |
| --- | --- | --- | --- | --- | --- |
| 1단계 - 문제 인식과 임팩트 맥락 잡기 | 왜 이 문제를 비즈니스로 다루어야 하는가를 사회·환경·경제 관점에서 명확히 함. 임팩트 비즈니스의 방향과 연결.  | - 이 문제가 사회적·환경적으로 어떤 손실을 내고 있는가?- 경제적 가치와 함께 창출될 수 있는가?- 문제를 잘못 잡았을 때의 비용은? | 임팩트 비즈니스 개념화, 목적-결과 매핑 | 문제 배경 설명 초안, 임팩트 관점 문제 정의 문장 | 이후 단계의 기준점. 여기서 모호하면 뒤가 다 흐릿해짐. |
| 2단계 - 현황 분석 | 현재 상태를 사실과 데이터로 붙잡아서 “어디가 문제인가”의 범위와 강도를 설정. | - 데이터 수집(통계, 인터뷰, 관찰)- 데이터 분석(심각성·발생 원인·영향 파악)- 문제의 지리·시간·대상 범위 명시 | 현황 분석, 기초 데이터 수집 템플릿 | 현황 분석 노트, 문제 영향도 요약(인구·지역·시간) | 이 단계가 약하면 뒤에서 자꾸 ‘추측형 문제’가 나옴. |
|   3단계 - 이해관계자 분석 | 실제로 이 문제에 얽힌 사람·기관의 관점과 니즈를 수집해 문제 정의를 현실화. 갈등·우선순위도 여기서 드러남. | - 이해관계자 식별(직접·간접)- 이해관계자 인터뷰·관찰 - 니즈·불편·제약 정리 - 영향 받는 정도에 따른 분류 | 이해관계자 맵, 인터뷰 가이드 | 이해관계자 목록, 니즈 테이블, 페인포인트 정리 | 임팩트 비즈니스는 수혜자와 지불자가 다른 경우가 많아 이 단계가 특히 중요함.  |
| 4단계 - 문제 구조화(원인-결과 파기) | 표면적 증상이 아니라 근본 원인을 보이게 해서 “어디를 개입해야 효과가 큰가”를 찾음. | - 문제나무 작성: 핵심 문제를 가운데 두고 위는 결과, 아래는 원인 - 5 Whys로 원인 깊이 파기 - 원인끼리 묶어 핵심 레버리지 포인트 찾기 | 문제나무 분석, 5 Whys 분석 | 문제 구조도, 핵심 원인 리스트, 개입 우선순위 | “증상 해결형”을 여기서 걸러냄. |
| 5단계 - 문제 정의 문서화·검증 | 분석 결과를 하나의 문서로 고정해서, 조직과 이해관계자가 같은 문제를 보게 함.  | - 문제 정의서 작성(배경, 현황, 이해관계자, 원인-결과, 영향) - 이해관계자와 공유 - 피드백 반영해 개정 | **문제 정의서(Problem Definition Document)** | 확정본 문제 정의서, 합의된 문제 진술문 | “일단 이걸로 간다”는 합의 장치. 이후 솔루션 설계의 기준. |
| 6단계 - 지속적 모니터링과 재정의 | 상황·데이터·이해관계자 변화에 따라 문제를 다시 조정해 비즈니스가 현실을 놓치지 않도록 함. | - 데이터 지속 수집 - 외부 환경 변화 모니터링 - 피드백 루프 운영 - 필요 시 문제 정의서 개정 | 모니터링 체크리스트, 피드백 루프 | 개정 이력, 최신 문제 정의서 버전 | “문제 정의는 한 번에 끝나지 않는다” |

### 단계별 해설

- **1단계 “이건 임팩트 비즈니스인가?” 질문하기**
    
    “매출만 나면 된다” VS “사회·환경 문제를 풀면서 경제성을 붙인다”
    
- **2단계 “이것이 진짜 문제인가?” 근거 찾기**
    
    데이터 수집(설문, 인터뷰, 현장 관찰)
    
    데이터의 구조화(심각성, 원인, 영향)
    
- **3단계 “문제에 대한 시각 왜곡 피하기”**
    
    임팩트 비즈니스의 문제는 이해관계자가 여럿이라 단일 시각으로 정의하면 왜곡된다!
    
    “식별 → 인터뷰 → 니즈 이해”의 3단계 접근 필요!
    
- **4단계 “문제나무와 5 Whys를 사용한 분석하기”**
    
    ```mermaid
        graph BT
            %% 3. Effects (결과/가지)
            subgraph "🌳 Effects (결과)"
                E1[계속되는 가정의 빈곤]
                E2[지역의 자원을 관리할 수 있는<br>사람이 거의 없음]
                E3[여성들이 지역을 떠나기 시작함]
                E4[청년들이 직업을 찾아<br>다른 지역으로 이주함]
            end

            %% 2. Core Problem (핵심 문제/줄기)
            subgraph "🌲 Core Problem (핵심 문제)"
                C[마을 수입원의 감소]
            end

            %% 1. Roots (원인/뿌리)
            subgraph "🌱 Roots (원인)"
                R1[대가족 - 자식들 각각이<br>상속받는 재산이 적어짐]
                R2[자본의 고갈]
                R3[교육 기회의 부족]
                R4[마을의 생활수준이 저하되고,<br>인재가 줄어듦]
            end

            %% 링크 연결 (원인 -> 핵심 문제)
            R1 --> C
            R2 --> C
            R3 --> C
            R4 --> C

            %% 링크 연결 (핵심 문제 -> 결과)
            C --> E1
            C --> E2
            C --> E3
            C --> E4

            %% 스타일링 (시각적 구분)
            classDef core fill:#ffe4e1,stroke:#b22222,stroke-width:2px;
            classDef effects fill:#e6f7ff,stroke:#1890ff,stroke-width:2px;
            classDef roots fill:#f6ffed,stroke:#52c41a,stroke-width:2px;

            class C core;
            class E1,E2,E3,E4 effects;
            class R1,R2,R3,R4 roots;
    ```

    “구조화된 그림”과 “깊이 파기 질문” 기법을 함께 사용해서, 사고의 폭과 깊이를 함께 추구!
