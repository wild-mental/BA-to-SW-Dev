# Product Requirement Document: AI 기반 창업 성장 플랫폼 v0.3

- **Owner**: Product & Engineering
- **Last Updated**: 2025-11-15

---

## 1. 개요·목표

### **1.1. 문제 정의 (Pain/Needs)**

- **(Why - 전략적 문제)** 창업 실패 원인의 42%가 '시장 수요 없음(No Market Need)'이며, 초기 창업가들은 정부 지원 사업계획서 작성에 평균 40시간 이상을 소요하며 이 중 40% 이상이 서류 심사에서 탈락하는 비효율을 겪고 있습니다.
- **(How - 실행적 문제)** 정부/은행 제출 시스템의 복잡한 양식으로 인해 Validator 실패율이 25%에 달하며, Notion/Jira 등 분절된 툴 사용으로 실행 계획과 재무 데이터 간 불일치가 발생하여 투자자 대응에 어려움을 겪습니다.

### **1.2. 목표 (Desired Outcomes)**

1.  **사업계획서 작성 시간 90% 단축**: 사용자가 30분 내에 정부/은행 제출 양식에 100% 호환되는 사업계획서를 완성하게 한다.
2.  **시장 수요 검증 강화**: 사용자가 자신의 아이디어가 가진 '시장 수요 부재' 리스크를 데이터 기반으로 90% 이상 식별하고 대응하게 한다.
3.  **제출 성공률 극대화**: 생성된 모든 사업계획서의 제출 호환성(Validator 통과율) 100%를 달성하고, 초안 작성부터 제출까지의 리드타임을 3일 이내로 단축한다.
4.  **실행-재무 데이터 정합성 확보**: 외부 툴 연동을 통해 실행 계획과 재무 예측 간의 불일치를 0건으로 유지한다.

### **1.3. 성공 지표 (KPIs)**

- **북극성 KPI**: **유효 사업계획서 제출률 (Validated Submission Rate)**
  - **정의**: 신규 가입 유저가 7일 내 '시장 위험 진단' 기능을 사용하고, 'Validator'를 통과한 사업계획서를 1회 이상 다운로드한 비율.
  - **기준선**: 0%
  - **목표값**: 10% (1Q)
  - **측정**: Daily / Weekly, Funnel 분석 (가입 → 진단 → Validator 통과 → 다운로드)
- **보조 KPI**:
  - **Time-to-Value**: 유저 가입 후 첫 사업계획서 완성까지 소요 시간 중앙값 < 30분
  - **제출 리드타임 중앙값**: 초안 생성부터 최종 제출(다운로드)까지 3일 이내
  - **시장 위험 진단 리포트 생성률**: 사업계획서 생성 유저의 '시장 위험 진단' 기능 사용률 > 25%
  - **실행-재무 불일치 리포트 생성률**: Notion/Jira 연동 유저의 주간 불일치 리포트 조회율 > 40%

## 2. 사용자와 페르소나

- **Segment 1 (과제 수행 그룹)**: 정부 지원 사업이나 은행 대출을 위해 사업계획서 제출이 시급한 예비창업가. (페르소나: 김예비, 박사장)
  - **Pain/Needs**: 복잡한 양식, 마감 스트레스, 특히 재무 파트 작성에 대한 막막함을 해소하고 싶다.
- **Segment 2 (가설 검증 그룹)**: 과거 '시장 수요 없음'으로 실패 경험이 있어, 데이터 기반의 객관적인 검증을 통해 실패를 반복하지 않으려는 재창업가. (페르소나: 최민혁, 한서윤)
  - **Pain/Needs**: 확증 편향에서 벗어나 아이디어의 실패 가능성을 사전에 진단받고, 투자자에게 논리적인 근거를 제시하고 싶다.
- **Segment 3 (워크플로우 통합 그룹)**: LivePlan 등 기존 툴의 데이터 파편화에 불만을 느끼는 스케일업 단계의 창업가. (페르소나: 이대표)
  - **Pain/Needs**: Notion/Jira의 실행 데이터와 재무 계획이 자동으로 동기화되어 일관성을 유지하고 싶다.

## 3. 사용자 스토리와 수용 기준 (AC)

### **Story 1: 과제 수행 그룹 (김예비)**
**As a** 정부 지원 사업 마감이 임박한 예비창업가,
**I want to** 내 아이디어와 기본 정보를 입력하여 심사 기준과 제출 규격에 100% 맞는 사업계획서와 재무 추정치를 빠르게 완성하고 싶다,
**so that** 형식 탈락 리스크 없이 안정적으로 자금을 확보할 수 있다.

- **AC 1 (양식 호환성 및 Validator)**
  - **Given**: 사용자가 '예비창업패키지' 표준 템플릿을 선택하고 필수 항목을 입력한 후,
  - **When**: '생성하기' 버튼을 클릭하면,
  - **Then**: 생성된 문서는 K-Startup 제출 시스템의 Validator를 99.9% 이상의 성공률로 통과해야 하며(p95 응답 ≤ 800ms), 실패 시 1초 내에 수정이 필요한 항목이 하이라이트 되어야 한다.
- **AC 2 (생성 속도)**
  - **Given**: 네트워크 지연이 없는 표준 환경에서,
  - **When**: 사용자가 사업계획서 생성을 요청하면,
  - **Then**: 로그인부터 최종 파일 다운로드까지의 전체 과정은 90th percentile 기준 30분 이내에 완료되어야 한다.

### **Story 2: 가설 검증 그룹 (최민혁)**
**As a** 과거 '시장 수요 없음'으로 실패 경험이 있는 재창업가,
**I want to** 내 아이디어의 핵심 가설을 입력하여 객관적인 데이터 기반의 시장 위험 진단 리포트를 받고 싶다,
**so that** 실패 공식을 반복하지 않을 것이라는 확신을 얻고 투자자를 설득할 수 있다.

- **AC 1 (리포트 품질 및 신뢰성)**
  - **Given**: 사용자가 '시장 위험 진단' 기능을 선택하고 핵심 가설을 3개 이상 입력하면,
  - **When**: 분석을 요청할 때,
  - **Then**: 리포트는 p95 기준 5분 이내에 생성되어야 하며, 모든 데이터 포인트는 신뢰할 수 있는 외부 출처(최소 1개 이상)를 명시해야 한다.
- **AC 2 (실행 가능성)**
  - **Given**: 사용자가 생성된 리포트를 검토하고,
  - **When**: 내용이 "실행 가능하다"고 판단하는지 평가할 때,
  - **Then**: 사용성 테스트(n=50) 결과, 70% 이상이 리포트 제언이 "실행 가능" 또는 "매우 실행 가능"하다고 5점 척도로 평가해야 한다.

### **Story 3: 워크플로우 통합 그룹 (이대표)**
**As a** 기존 툴의 데이터 파편화 문제를 겪는 창업가,
**I want to** Notion/Jira에 기록된 실행 계획과 재무 계획을 자동으로 비교하여 불일치 항목을 확인하고 싶다,
**so that** 실행과 재무 데이터의 정합성을 항상 유지할 수 있다.

- **AC 1 (데이터 동기화 및 리포팅)**
  - **Given**: 사용자가 Notion/Jira API를 연동(OAuth)하고,
  - **When**: 주간 자동 동기화가 실행될 때,
  - **Then**: p95 기준 60초 이내에 동기화가 완료되어야 하며, 불일치 항목이 발견될 경우 사용자에게 diff 리포트가 이메일로 발송되어야 한다. (동기화 성공률 월 ≥ 99.0%)

## 5. 비기능 요구사항 (NFR)

- **성능**:
  - **p95 응답 ≤ 목표**: Validator ≤ 800ms, 문서 자동작성 ≤ 4s, 진단 리포트 ≤ 5s, 싱크 diff 리포트 ≤ 60s
  - **모니터링**: APM을 통해 각 API 엔드포인트의 p95, p99 응답 시간 및 처리량 로깅. 5분 이상 임계치 초과 시 P2 알림.
- **신뢰성**:
  - **월 가용성 ≥ 99.9%**, **백엔드 오류율 ≤ 0.5%**, **동기화 성공률 ≥ 99.0%**
  - **RPO ≤ 1h, RTO ≤ 30m**
  - **모니터링**: Health Check API(1분 주기) 응답 실패 5회 연속 시 P1 알림. 5xx 오류율이 5분간 1% 초과 시 P2 알림.
- **보안/비용**:
  - **보안**: 사용자 데이터 전송(TLS 1.2+) 및 저장(AES-256) 암호화. PIPA 준수. RBAC, 감사 로그 1년 보관.
  - **비용**: 문서 1건 자동작성·검증 변동비 ≤ $0.10. 1인당 월평균 인프라 비용 < $2.00
  - **모니터링**: 정기 보안 취약점 점검(연 1회), SAST 자동화. 월별 클라우드 비용 임계치 80% 도달 시 알림.

## 6. 데이터·인터페이스 개요

- **핵심 엔터티**: `User`, `BusinessPlan`, `FinancialStatement`, `MarketRiskReport`, `Hypothesis`, `ValidatorRule`, `SyncConnector`
- **주요 필드**:
  - `BusinessPlan`: `templateType`, `companyInfo`, `marketAnalysis`, `validatorScore`
  - `MarketRiskReport`: `tamSamSomData`, `competitorAnalysis`, `riskFactors`
  - `SyncConnector`: `sourceType` (Notion/Jira), `schemaMap`, `lastRunAt`, `successRate`
- **외부/내부 API**:
  - `POST /documents`: 입력(템플릿/질문 답변) → 출력(초안 문서, validatorScore)
  - `POST /validator/run`: 입력(documentId) → 출력(pass/fail, rejectReasons[])
  - `POST /pmf/report`: 입력(jtbdCards[]) → 출력(risks[], aosDosMatrix, coverage)
  - `POST /sync/run`: 입력(connectorId) → 출력(diffReport, mismatchCount)
  - **제약**: 외부 API 호출 실패 시(Timeout > 5s) Circuit Breaker 및 Fallback 로직 필수. 요청 크기 ≤ 10MB, Rate-Limit 60rpm/user.

## 7. 범위(In/Out), 리스크·가정·의존성

- **In Scope**:
  - 시장 위험 진단 기반 AI 사업계획서 초안 자동 생성 (PDF, DOCX)
  - 국내 10대 초기 창업 지원사업 템플릿 및 Validator
  - 3개년 재무제표(추정 손익계산서, 현금흐름표) 자동 생성
  - Notion/Jira 연동을 통한 실행-재무 불일치 검증 리포트
- **Out of Scope**:
  - 법률 자문, 법인 등기, 회계 SaaS 양방향 통합
  - 실시간 다중 사용자 협업 편집
  - 정부 포털 직접 제출, 투자자 매칭
- **리스크 및 대응**:
  1.  **(기술) AI 재무 추정 신뢰도 부족**: 심사위원이 납득할 논리와 정확도 확보 실패 → **대응**: 근거 링크 및 계산식 출처 명시, 현직 VC 심사역 대상 블라인드 테스트 통과 의무화.
  2.  **(시장) 단기 과제 해결 후 이탈**: 사업계획서 생성 후 '가설 검증' 및 '데이터 연동' 가치로 전환 실패 → **대응**: 온보딩 과정에서 가설 검증 기능의 중요성 강조, 데이터 연동 시 추가 크레딧 제공.
  3.  **(보안) 민감 아이디어에 대한 AI 불신**: 데이터 보안 정책에도 불구, 사용자가 아이디어 입력을 꺼릴 가능성 → **대응**: 데이터 분리 저장 및 비식별화 처리, 국내 리전 옵션 제공, 보안 규정 준수 인증 획득.
- **가정 및 의존성**:
  - **가정**: AI가 생성한 재무 추정치는 현직 VC 심사역 대상 블라인드 테스트에서 '납득 가능' 평가를 70% 이상 획득할 것이다.
  - **의존성**: 외부 LLM API 및 Notion/Jira API의 안정성(가용성 99.5% 이상) 및 정책 변경 리스크.

## 8. 실험·롤아웃·측정

- **베타 채널**: 재창업가 커뮤니티(n=100), 대학 창업센터, 세무사 파트너 대상 비공개 베타 4주 진행.
- **실험 1: '자동완성+Validator'가 제출 통과율을 개선하는가?**
  - **가설**: 우리 솔루션은 수기 작성 대비 1차 통과율을 20%p 이상 개선한다.
  - **측정**: A/B 테스트 (A: 솔루션, B: 수기 작성), n≥100 문서의 validator 통과율, 형식/누락 사유 리젝 감소율 비교.
- **실험 2: '시장 위험 진단 리포트'가 의사결정에 영향을 미치는가?**
  - **가설**: 리포트는 사용자의 비즈니스 계획에 실질적 영향을 미친다.
  - **측정**: A/B 테스트 (A: 리포트 제공, B: 미제공), n≥200 유저의 2주 후 '사업 가설 수정' 응답률 비교.
- **경쟁 대안 벤치마크**:
  - **대상**: LivePlan (글로벌 SaaS), 1:1 컨설팅(고비용 대안)
  - **계획**: ①LivePlan 대비 국내 템플릿 호환율 및 Validator 통과율 비교. ②컨설팅 결과물과 우리 솔루션 결과물을 현직 심사역 3명에게 블라인드 평가 의뢰.
