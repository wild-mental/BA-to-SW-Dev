# Product Requirement Document: AI 기반 창업 Co-Pilot v0.1

- **Owner**: Product Team
- **Last Updated**: 2025-11-13

---

## 1. 개요·목표

- **문제 정의 (Pain/Needs)**:
  - **Pain 1**: 초기 창업가들은 정부 지원 사업계획서 작성에 평균 40시간 이상을 소요하며, 이 중 40% 이상이 서류 심사에서 탈락하는 비효율을 겪고 있습니다. (실패 KPI: 서류 탈락률 > 40%)
  - **Pain 2**: 창업 실패 원인의 42%가 '시장 수요 없음'임에도 불구하고, 창업가들은 객관적인 데이터 없이 아이디어를 검증하여 '성급한 확장' 실패(74%)를 반복합니다. (실패 KPI: '시장 수요 없음'으로 인한 실패율 = 42%)

- **목표 (Desired Outcomes)**:
  1.  **사업계획서 작성 시간 90% 단축**: 사용자가 30분 내에 정부/은행 제출 양식에 100% 호환되는 사업계획서를 완성하게 한다.
  2.  **시장 수요 검증 강화**: 사용자가 자신의 아이디어가 가진 '시장 수요 부재(No Market Need)' 리스크를 데이터 기반으로 90% 이상 식별하고 대응하게 한다.
  3.  **재무 예측 자동화**: 복잡한 재무 지식 없이도 투자(IR) 및 대출용 3개년 재무제표를 10분 내에 자동 생성하게 한다.

- **성공 지표 (KPIs)**:
  - **북극성 KPI**: **활성 유저의 '제출 준비 완료' 사업계획서 생성률**
    - **정의**: 신규 가입 후 7일 내 '정부/은행 제출용 템플릿'을 선택하여 최종 문서를 1회 이상 다운로드한 유저의 비율
    - **기준선**: 0%
    - **목표값**: 15% (1Q)
    - **측정 주기**: Daily / Weekly
  - **보조 KPI**:
    - **Time-to-Value**: 유저 가입 후 첫 사업계획서 완성까지 소요 시간 중앙값 < 30분
    - **유료 전환율**: 무료 체험 종료 후 유료 플랜 전환율 > 5%
    - **시장 위험 진단 리포트 생성률**: 사업계획서 생성 유저의 '시장 위험 진단' 기능 사용률 > 25%

## 2. 사용자와 페르소나

- **핵심 페르소나**:
  - **Segment 2 (과제 수행 그룹)**: 정부 지원 사업이나 은행 대출을 위해 사업계획서 제출이 시급한 예비창업가. (페르소나: 김예비, 박사장)
    - **Pain/Needs**: 복잡한 양식, 특히 재무 파트 작성에 대한 막막함과 불안감을 해소하고 싶다.
  - **Segment 3 (가설 검증 그룹)**: 과거 '시장 수요 없음'으로 실패 경험이 있어, 데이터 기반의 객관적인 검증을 통해 실패를 반복하지 않으려는 재창업가. (페르소나: 최민혁, 한서윤)
    - **Pain/Needs**: 확증 편향에서 벗어나 아이디어의 실패 가능성을 사전에 진단받고 싶다.

## 3. 사용자 스토리와 수용 기준(AC, Acceptance Criteria)

### **Story 1: 과제 수행 그룹 (김예비)**
**As a** 정부 지원 사업 마감이 임박한 예비창업가,
**I want to** 내 아이디어와 기본 정보를 입력하여 심사 기준에 맞는 사업계획서와 재무 추정치를 빠르게 완성하고 싶다,
**so that** 탈락 리스크 없이 안정적으로 자금을 확보할 수 있다.

- **AC 1 (양식 호환성)**
  - **Given**: 사용자가 '예비창업패키지' 표준 템플릿을 선택하고,
  - **When**: 필수 항목을 모두 입력한 후 '생성하기' 버튼을 클릭하면,
  - **Then**: 생성된 문서는 K-Startup 제출 시스템의 형식 검증을 99.5% 이상의 성공률로 통과해야 한다.
- **AC 2 (생성 속도)**
  - **Given**: 네트워크 지연이 없는 표준 환경에서,
  - **When**: 사용자가 사업계획서 생성을 요청하면,
  - **Then**: 로그인부터 최종 파일 다운로드까지의 전체 과정은 90th percentile 기준 30분 이내에 완료되어야 한다.
- **AC 3 (재무 추정 정확도)**
  - **Given**: 사용자가 업종(예: SaaS)과 핵심 변수(예: 월 구독료)를 입력하면,
  - **When**: 시스템이 3개년 추정 손익계산서를 생성할 때,
  - **Then**: 주요 항목(매출, 영업이익)의 오차는 주니어 컨설턴트 분석 결과 대비 ±15% 이내여야 한다.

### **Story 2: 가설 검증 그룹 (최민혁)**
**As a** 과거 '시장 수요 없음'으로 실패 경험이 있는 재창업가,
**I want to** 내 아이디어의 핵심 가설을 입력하여 객관적인 데이터 기반의 시장 위험 진단 리포트를 받고 싶다,
**so that** 실패 공식을 반복하지 않을 것이라는 확신을 얻을 수 있다.

- **AC 1 (리포트 생성 속도)**
  - **Given**: 사용자가 '시장 위험 진단' 기능을 선택하고 핵심 가설을 3개 이상 입력하면,
  - **When**: 분석을 요청할 때,
  - **Then**: 리포트는 95th percentile 기준 5분 이내에 생성되어야 한다.
- **AC 2 (데이터 신뢰성)**
  - **Given**: 리포트가 생성되었을 때,
  - **When**: 사용자가 리포트 내의 시장 규모나 경쟁사 데이터를 확인하면,
  - **Then**: 각 데이터 포인트는 최소 1개 이상의 신뢰할 수 있는 외부 출처를 명시해야 한다.
- **AC 3 (실행 가능성)**
  - **Given**: 사용자가 생성된 리포트를 검토하고,
  - **When**: 내용이 "실행 가능하다"고 판단하는지 평가할 때,
  - **Then**: 사용성 테스트(n=50) 결과, 70% 이상이 리포트 제언이 "실행 가능" 또는 "매우 실행 가능"하다고 5점 척도로 평가해야 한다.

## 4. 기능 요구사항(Functional)

| Priority | Feature ID | 기능명 | 근거 (대안 대비 가치) |
| :--- | :--- | :--- | :--- |
| **Must** | F1 | AI 사업계획서 자동 생성 | 수동 작업 대비 작성 시간을 80배 단축하여 가장 시급한 Pain 해결. |
| **Must** | F2 | 국내 창업 환경 맞춤 템플릿 | 글로벌 SaaS가 제공 못하는 국내 양식 90% 이상 호환. |
| **Must** | F3 | AI 재무 추정 자동화 | 1:1 컨설팅 대비 95% 비용 절감 및 시간 단축. |
| **Must** | F4 | 시장 위험 진단 리포트 | '시장 수요 없음' 실패 원인을 직접 타겟하는 핵심 차별화 기능. |
| **Should** | F5 | '쉬운 용어' 모드 & 튜토리얼 | 비전문가 페르소나의 초기 이탈률 30% 감소 기대. |
| **Could** | F6 | 가설-검증 관리 보드 | 진성 유저(가설 검증 그룹)의 리텐션 20% 향상 기대. |
| **Won't** | F7 | 외부 생산성 도구 연동 (Jira, Notion) | MVP 단계에서는 핵심 가치(진단, 실행)에 집중. |

## 5. 비기능 요구사항(NFR, Non-Functional Requirement)

- **성능**:
  - **p95 응답 ≤ 3,000 ms** (핵심 API: 사업계획서 생성, 재무 추정, 리포트 분석)
  - **모니터링**: APM을 통해 각 API 엔드포인트의 p95, p99 응답 시간 및 처리량 로깅. 5분 이상 임계치 초과 시 P2 알림.
- **신뢰성**:
  - **월 가용성 ≥ 99.8%**, **오류율 ≤ 0.5%**
  - **모니터링**: Health Check API(1분 주기) 응답 실패 5회 연속 시 P1 알림. 5xx 오류율이 5분간 1% 초과 시 P2 알림.
- **보안/비용**:
  - **보안**: 사용자 입력 데이터 전송(TLS 1.2+) 및 저장(AES-256) 암호화. PIPA 준수.
  - **비용**: 1인당 월평균 인프라 비용 < $2.00
  - **모니터링**: 정기 보안 취약점 점검(연 1회), SAST 자동화. 월별 클라우드 비용 임계치 80% 도달 시 알림.

## 6. 데이터·인터페이스 개요

- **핵심 엔터티**: `User`, `BusinessPlan`, `FinancialStatement`, `MarketRiskReport`, `Hypothesis`
- **주요 필드**:
  - `BusinessPlan`: `templateType`, `companyInfo`, `itemInfo`, `marketAnalysis`
  - `MarketRiskReport`: `tamSamSomData`, `competitorAnalysis`, `riskFactors`
- **외부 API**:
  - **입력**: 공공데이터포털(기업 정보, 시장 통계), 외부 LLM API
  - **제약**: 외부 API 호출 실패 시(Timeout > 5s)를 대비한 Circuit Breaker 및 Fallback 로직 필수.

## 7. 범위(In/Out), 리스크·가정·의존성

- **In Scope**:
  - AI 기반 사업계획서 초안 자동 생성 (PDF, DOCX)
  - 국내 5대 초기 창업 지원사업 템플릿
  - 3개년 재무제표(추정 손익계산서, 현금흐름표) 자동 생성
  - 시장 데이터 기반 '시장 수요 부재' 위험 진단 리포트
- **Out of Scope**:
  - 법률 자문, 법인 등기 문서 생성
  - 실시간 다중 사용자 협업 편집
  - 정부 포털 직접 제출, 투자자 매칭
- **리스크**:
  1.  **(기술) AI 재무 추정 신뢰도 부족**: 심사위원이 납득할 논리와 정확도 확보 실패 가능성.
  2.  **(시장) 단기 과제 해결 후 이탈**: 사업계획서 생성 후 '가설 검증' 가치로 전환 실패.
  3.  **(보안) 민감 아이디어에 대한 AI 불신**: 데이터 보안 정책에도 불구, 사용자가 아이디어 입력을 꺼릴 가능성.
- **가정 및 의존성**:
  - **가정**: AI가 생성한 재무 추정치는 현직 VC 심사역 대상 블라인드 테스트에서 '납득 가능' 평가를 70% 이상 획득할 것이다.
  - **의존성**: 외부 LLM API의 안정성(가용성 99.5% 이상) 및 정책 변경 리스크.

## 8. 실험·롤아웃·측정

- **베타 채널**: 재창업가 커뮤니티(n=100) 대상 비공개 베타 4주 진행
- **실험 1: 사업계획서 작성 시간 단축 검증**
  - **가설**: 우리 솔루션은 수동 작업 대비 작성 시간을 50배 이상 단축한다.
  - **측정**: 사용성 테스트(n=20), A그룹(솔루션) vs B그룹(수동) 과제 완료 시간 및 CSAT 비교.
- **실험 2: 시장 위험 진단 리포트 유효성 검증**
  - **가설**: 리포트는 사용자의 비즈니스 계획에 실질적 영향을 미친다.
  - **측정**: A/B 테스트(n=200), 리포트 제공 그룹(A)과 미제공 그룹(B)의 2주 후 '사업 가설 수정' 응답률 비교.
- **경쟁 대안 벤치마크**:
  - **대상**: LivePlan (글로벌 SaaS), 1:1 컨설팅(고비용 대안)
  - **계획**: ①LivePlan 대비 국내 템플릿 호환율 및 재무 용어 정확도 비교. ②컨설팅 결과물과 우리 솔루션 결과물을 현직 심사역 3명에게 블라인드 평가 의뢰.
