## GPT5 Value Proposition Sheet — 초보 예비창업자용 (Problem–Solution Fit)

문서 목적: 처음으로 본격적인 신규 사업을 기획하는 예비창업자를 위한 가치 제안서. 아래 내용은 TAM–SAM–SOM 분석, 고객 페르소나/여정지도, JTBD 가상 인터뷰(V2) 결과를 통합하여 작성했으며, 작성 방법은 `8-3_ValueProposition_Sheet_작성방법.md`를 기준으로 구성했습니다.

---

### 1) 페르소나 및 CJM 기반 핵심 문제(Pain)·니즈(Needs)

- 핵심 타깃: 초보 예비창업자(첫 창업), 기술·AI 비전공 비중 높음
  - Q2 과제 수행형: 정부지원(예창패)·은행대출·경진대회 등 “제출/통과”가 당면 과제
  - Q1 학습/검증형: “지난 실패를 반복하지 않기” 위해 데이터 기반 PMF 검증을 선호
- 대표 Pain
  - 사업계획서·재무 추정이 가장 큰 장벽(용어·양식·숫자 공포)
  - ‘시장 수요 없음’ 재발 우려, 계획(UIs)과 실행·재무의 단절
  - 보안/데이터 유출 불안(딥테크·민감정보)
- 대표 니즈
  - 한글 기반 쉬운 단계별 안내(Wizard)와 정부/은행 양식 100% 호환
  - 핵심 변수 몇 개로 전문가 수준 재무 자동화
  - 가설-시장-재무를 한 화면에서 검증·연동(PMF 대시보드)

출처 요약: `6_고객_페르소나_스펙트럼&여정지도.md`, `8-2_JTBD_가상인터뷰_결과지_V2.md`

### 2) JTBD 기반 고객 상황별 목표(Job) 요약

- 과제 통과(Job to pass the test)
  - 예창패/은행/경진대회 제출용 사업계획서·재무제표를 빠르고 정확하게 완료
- 실패 회피(Job to avoid failure)
  - 주관 아닌 데이터·프레임워크로 PMF를 사전 검증해 ‘성급한 확장’ 재발 방지
- 장벽 극복(Job to overcome blockers)
  - 보안(설치형), 사용성(쉬운 한글 Wizard), 재무 자동화(핵심 변수 입력)로 이탈 지점 해소

대표 카드: 김예비(예창패), 박사장(대출), 박은숙(사용성), 오다빈(재무), 최민혁(PMF) — `8-2_JTBD_V2.md`

### 3) 고객 Desired Outcomes (우선순위 근거)

JTBD AOS/DOS 표(요약)에서 초보 창업자와 직접 연관된 상위 Outcomes:

| Outcome | AOS | DOS | 근거(인용) |
| --- | --- | --- | --- |
| 정부 지원 양식 100% 호환 계획서 완성 | 4.0 | 3.2 | “합격하는 계획서가 필요” (김예비) |
| 단계별 안내(Wizard)로 끝까지 완성 | 4.0 | 3.2 | “포기하지 않게 도와주는 친절함” (박은숙) |
| 은행 심사 기준 재무 자동 생성 | 3.2 | 2.4 | “은행에서 원하는 건 따로 있다” (박사장) |
| 핵심 변수 입력만으로 재무 자동화 | 3.2 | 2.4 | “엑셀만 켜면 머리 하얘진다” (오다빈) |
| 내 아이템 실패 위험 사전 검증 | 3.0 | 2.7 | “적신호 1개가 더 중요” (최민혁) |

### 4) 우리 솔루션의 핵심 제안(Value Proposition)

- 한 줄 정의: “초보 창업자를 위해 정부·은행 양식에 맞춘 사업계획서와 전문가 수준 재무/PMF 진단을 한글 Wizard로 빠르게 완성해 ‘통과’와 ‘실패 회피’를 동시에 달성”
- 핵심 가치 묶음
  - Pass the Gate: 예창패/은행/경진대회 제출용 템플릿 100% 호환, HWP/PDF 원클릭 내보내기
  - Validate Before Build: 가설-고객-시장지표와 재무를 연동한 PMF·IR 대시보드
  - Overcome Blockers: 한글 Wizard, 쉬운 모드↔전문가 모드, 핵심 변수 기반 재무 자동화
  - Trust & Security: On-prem/Private Cloud 옵션으로 민감 데이터 보호

### 5) 기존 대안(Competitor / Substitute)

- 해외 SaaS: LivePlan, Upmetrics, VentureKit(영어 중심/현지화 미흡, 정부/은행 양식 호환 낮음)
- 수기 툴: Excel + Notion + 검색 템플릿(일관성/연동/신뢰 부족, 완성도 한계)
- 외주 컨설팅: 고비용(수백만 원), 반복·학습 불가, 의존성 증가

### 6) 차별적 가치(Differentiators)

- 국내 제출 양식 정합성: 예창패/은행/경진대회 기준 ‘검증된’ 템플릿·산출물 자동화
- 한국어 초보자 UX: 쉬운 한글 Wizard + 용어 쉬운말 툴팁 + 목적별 온보딩
- PMF↔재무↔IR 연동: “가설→지표→매출/LTV” 논리 사슬을 숫자로 방어 가능
- 실행-재무 동기화: Notion/Jira 연동으로 ‘유령 계획서’ 방지
- 보안 우선: On-prem/Private Cloud 옵션, 보안 백서·NDA, 국내 인증 로드맵

## 성공 지표(초기 90일)

- Activation: 첫 세션 내 완성률(초안→제출 가능 산출물) 40%+

## (부록) 빠른 체크리스트 — 초보 예비창업자

- 오늘 할 일
  - 1) 목적 선택: 제출(예창/은행/대회) / 검증(PMF/IR)
  - 2) Wizard 시작: 한글 단계별 입력 → 기본 초안 생성
  - 3) 핵심 변수 5개로 재무 자동 생성 → 제출 양식 미리보기
  - 4) 필요 시 ‘전문가 모드’로 보강 → PDF/HWP 내보내기
- 이번 주 목표
  - 1) ‘가설-지표-매출’ 연결 점검(대시보드)
  - 2) 심사/투자 FAQ 대비 표 작성
  - 3) 다음 주 실행 가설 1건 세팅(제품-시장 학습 루프)
